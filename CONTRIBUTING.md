The following is a set of guidelines for contributing to this project. These are mostly guidelines, not rules. Use your best judgment, and feel free to propose changes to this document in a pull request.

## How Can I Contribute?

### Reporting Bugs
This section guides you through submitting a bug report.
- Check if the bug has already been reported in the Issues tab.
- Use the **Bug Report** issue template to report a new bug.
- Include as many details as possible (OS, browser, steps to reproduce, console errors).

### Suggesting Enhancements
This section guides you through submitting an enhancement suggestion.
- Check if the feature has already been requested in the Issues tab.
- Use the **Feature Request** issue template to suggest an enhancement.
- Explain how this enhancement would be useful to most users.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. Ensure you have installed all dependencies via `npm install`.
3. If you've added code that should be tested, add tests.
4. If you've changed APIs, update the documentation.
5. Create a Pull Request using the provided PR template.
6. Make sure your PR is focused on a single change.

## Development Setup
Check the `README.md` for detailed instructions on how to set up the environment with Docker, or how to run the services individually in development mode.

## Code Style
1. We use React (+ Vite) on the frontend and Express (Node.js) on the backend.
2. We enforce standard TypeScript styling. 
3. Make sure your code is clean and readable before submitting a PR.
